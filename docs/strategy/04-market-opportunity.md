## Executive Summary

The Infrastructure-as-Code (IaC) tooling market is growing rapidly as organisations
adopt cloud-native architectures. Static analysis and policy enforcement tools
represent a significant and underserved segment.

## Total Addressable Market

- **IaC tooling market:** Estimated $2B+ by 2027
- **DevSecOps tooling:** Adjacent and overlapping, $4B+
- **Target segment:** Developer-first, open-source-led IaC analysis tools

## Competitive Landscape

| Competitor | Strengths | Weaknesses |
|------------|-----------|------------|
| Checkov | Broad rule set | Limited AI suggestions |
| TFLint | Terraform-specific | Narrow scope |
| Semgrep | General purpose | Not IaC-focused |
| infralint | AI-powered, open-source | Early stage |

## Key Differentiators

1. AI-powered rule suggestions reduce time-to-value.
2. Apache 2.0 licence removes commercial adoption barriers.
3. Extensible plugin architecture supports custom rule authoring.

## Go-to-Market Strategy

- Community-led growth via open-source releases.
- Developer advocacy through blog posts, conference talks, and tutorials.
- Enterprise tier with SLA support and private rule registries.
