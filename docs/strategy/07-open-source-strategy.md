# Open-Source Strategy

> **Status update (2026-05-08):** This project has been renamed **infralint** and
> released under Apache 2.0. See the root [README](../../README.md) and
> [CHANGELOG](../../CHANGELOG.md). The strategic analysis below remains valid;
> product/positioning decisions are now reflected in the current codebase.

## Objectives

1. Build a thriving contributor community around infralint.
2. Establish infralint as the default IaC linter for cloud-native teams.
3. Create a sustainable commercial layer without fragmenting the open-source core.

## License Choice: Apache 2.0

Apache 2.0 was chosen because it:

- Permits use in commercial products without copyleft obligations.
- Is widely trusted by enterprise legal and procurement teams.
- Encourages contribution from both individuals and organisations.

## Community Building

| Initiative | Goal | Timeline |
|------------|------|----------|
| Good-first-issue programme | Onboard new contributors | Ongoing |
| Monthly community call | Align roadmap with community | Monthly |
| Public roadmap on GitHub | Transparency and trust | Live now |
| Contributor recognition | Retention and motivation | Each release |

## Commercial Strategy

- **Open core:** Core linting engine remains fully open-source.
- **Enterprise add-ons:** Private rule registries, SSO, audit logs, SLA support.
- **Hosted SaaS:** Optional managed service for teams that prefer zero-ops.

## Success Metrics

- GitHub stars growth (target: 1,000 in 6 months post-launch).
- Number of external contributors per release cycle.
- Community forum response time (target: < 24 hours).
- Enterprise pipeline conversion rate from open-source users.
